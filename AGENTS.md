## 開発要件
 - 基本的なGoのベストプラクティスに従うこと
 - `main.go`でGraceful Shutdownを実現すること
 - 不足しているパッケージは、`go get`コマンドで逐次取得してください。ただし、標準ライブラリで実行することが可能な物事にし対して、不必要に外部のライブラリを使用しないでください。依存関係は最小限に保つという方針です。
 - なるべくテストを書いてください

## 機能
 - このリポジトリは「yt-dlpを使用して動画をダウンロードし、それをオブジェクトストレージにアップロードしてデータベースに記録し、いつでもブラウザで見れるようにする」というプロジェクトのマイクロサービスAPIサーバーです
 - すべての動画は個人利用のためにアーカイブされるものであり、指定されたユーザーのIDを主キーとしてMongoDBに書き込んでください。データベースの設計は任せます。
 - 動画のメタデータ（タイトルやチャンネルIDなど）は公式のYouTube APIを使用して取得し、MongoDBに書き込んでください。
 - yt-dlp実行時、`yt-dlp -f "bestvideo[vcodec*=av01]+bestaudio[acodec*=opus]/bestvideo[vcodec*=vp9]+bestaudio[acodec*=opus]"`を使用してAV1かVP9での取得を試みてください。フォーマットが見つからないというエラーをyt-dlpが返した場合のみ、bestvideoとbestaudioのみでコーデックを指定せず再試行してください。

## API
 - `/v1/queue/add`で動画のURLをPOSTされるので、YouTube APIを使用し動画の情報を取得でき、yt-dlpの起動に成功した時点で200と共にデータベースでの動画IDを返してください。データベースにはstatusがpendingであることを書き込んでください。ダウンロードに成功したら、オブジェクトストレージにアップロードしてそのURLをデータベースに書き込んでください。goroutineを使用してダウンロードとアップロードを行うこと。
 - yt-dlpでのダウンロードが失敗した場合、statusをfailedにしてエラーをコンソールにログしてください。
 - `/v1/queue/:id/status`にGETされた場合、ジョブのステータスを返してください。
